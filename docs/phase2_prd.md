# Luxury Resort Finder — Phase 2 PRD

## Objective
Turn the current resort directory into a decision workflow that helps users save, compare, and remember resorts with minimal friction.

## Problem
The app currently helps users browse resorts, but it does not yet own the decision-making workflow.

## Phase 2 Scope
1. Shortlist: save and unsave resorts.
2. Compare: compare selected resorts side by side.
3. Memory: persist user preferences and recent intent.

## Success criteria
- Users save at least one resort.
- Users compare at least two resorts.
- Preferences persist across visits.
- The workflow feels useful without needing chat-first AI.

## Not in scope
- Full concierge automation.
- Agent orchestration.
- Booking engine.
- Partner dashboards.
- Overbuilt recommendation scoring.

## Implementation order
1. Save/unsave behavior.
2. Shortlist page or drawer.
3. Compare UI.
4. Persisted preference state.
5. Tie shortlist and memory into one flow.

## Defensibility
The product becomes harder to replace when it owns user memory, decision history, and comparison state over time.
